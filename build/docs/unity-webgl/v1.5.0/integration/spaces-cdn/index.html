Spaces CDN

CDN Setup​

If Your Games Does Not Use Addressables by Default​

  1. Go to Windows -> Asset Management -> Addressables Groups -> Click on Create Addressables Settings

  1. Ignore the Legacy Asset Bundles, to keep it as a separate module from addressables
  2. Create Addressable Settings. Right Click on the Default Local Group and Rename it to SpacesGroup.

  1. Click on SpacesGroup, on the right side you will see an inspector window pop-up
    1. Turn on Use Asset Bundle Cache
    2. Set Cache Behaviour into Clear When When New Version Loaded
    3. Click on Inspect Top Level Settings
    4. Turn on Build Remote Catalog
    5. Click on Manage Profiles

  1. Set Remote into Custom
    1. Set Remote.BuildPath as ServerData/[BuildTarget]
    2. Set Remote.LoadPath as {SpacesLoader.HostURL}
    3. Close the Profile Window

  1. Under Catalog set Build and Load Paths as Remote

  1. Now in Double Click on SpacesGroup in Addressables Group Window, In Content Packing & Loading set Build & Load Paths to Remote

If Your Games Already Uses Addressables​

  1. Open Addressables Groups
  2. Right Click and Create New Group and then Select Packed Assets

  1. Rename it to SpacesGroup
  2. Click on Inspect Top Level Settings and Click on Manage Profiles

  1. Click on Create at the TopLeft Corner
  2. Create Variable (All Profiles)

  1. Set the Prefix Name as Spaces Load Path
  2. Set Load Path Value as {SpacesLoader.HostURL} and close the window

  1. Now Double Click on SpacesGroup in Addressables Group Window, In Content Packing & Loading set Build Path to the Path of your primary Catalog Build Path & Load Path to Spaces Load Path which you currently set from variables

  1. Turn on Use Asset Bundle Cache
  2. Set Cache Behaviour into Clear When When New Version Loaded

  1. Click on Inspect Top Level Settings and turn on Build Remote Catlog
  2. Set Catalog Build & LoadPaths to custom. Save your current Load Path elsewhere, you will need it in the next step. Do not alter the Build Path, set the Load Path to Spaces Load Path.

  1. Now Call SpacesController.SetDefaultHostUrl(string loadPath); before where ever you are calling await SpacesController.Load(); and pass the Load Path of your Catalog that you had saved in the previous step. If you are generating the Catalog Load Path in Runtime, make sure you fetch the LoadPath and pass it in SpacesController.SetDefaultHostUrl(string loadPath);
    SpacesController.SetDefaultHostUrl("your-previous-default-addressables-group-load-path");
    await SpacesController.Load();

CDN Asset-Linking​

  1. In Assets Folder under Spaces Directory, Select all the files Drag and Drop all the files into SpacesGroup Tab

  1. Now we have to rename all the in SpacesGroup to their respective file names. For example Assets/Spaces/Assets/Coin/SpacesCoinMesh.mesh into `SpacesCoinMesh etc.
  2. Rename all the Assets

  1. Select all the Sprites, Sprite Atlas & Textures in SpacesGroup and Turn on Use Crunch Compress and Click Apply
  2. Select all the assets in SpacesGroup and in the Labels tab, click on it and type spacesGroup and hit enter.
  3. Finally the SpacesGroup will look something similar like this

  1. Now go to Build under Addressables Groups and click on New Build -> Default Build Scripts.

  1. Now go to file explorer, to your build path. Send us all the files present in the WebGL folder

Cool, Integration is Done.

Running Campaigns Workflow.​

  1. Open the Unity Project in WebGL platform
  2. We will send you a list of files that you need to Copy and replace at the Assets folder under Spaces Directory
  3. After that in Addressables Groups click Build and click on Update a previous Build

NOTE: NEVER click on update New Build after the first time integration

  1. Now go to file explorer, to your build path. Send us all the files present in the WebGL folder

And That’s it.
